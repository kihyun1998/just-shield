# 사실 기반 판정 원칙: 빌드를 깨뜨리는 경고에 추측을 허용하지 않음

오탐이 누적되면 사용자가 도구를 꺼버리므로, 빌드 실패(최고 심각도) 판정은 검증 가능한 사실에서만 나오도록 구조적으로 제한한다. 네 가지 원칙: ① 빌드를 깨는 판정은 사실 규칙(R1, R4, R5, R8, R9 등 — 문법적/조회 가능한 사실)만 내릴 수 있고, 휴리스틱 규칙(R2 타이포스쿼팅, R3 패턴 매칭)은 단독으로는 안내 등급까지만. ② 휴리스틱을 상위 등급으로 승격하려면 교차 검증(예: 의심 저장소의 생성일·인지도와 원본의 대비)으로 증거를 모아야 한다. ③ 모든 경고는 파일·행·사실·출처를 근거로 제시한다. ④ 정확도는 주장이 아니라 측정 — 오탐 코퍼스(유명 저장소의 실제 워크플로, 최고 등급 오탐 0 요구)와 미탐 코퍼스(TeamPCP 시나리오 재현 픽스처, 미탐 0 요구)를 릴리스 게이트로 둔다.

트레이드오프: "더 많이 잡기"를 포기하고 "확실하게 잡기"를 선택한 것이다. 미래에 휴리스틱 규칙을 늘려 탐지 폭을 키우자는 제안이 오면, 이 ADR이 "그 규칙은 빌드를 깰 수 없다"는 한계선을 지킨다.
