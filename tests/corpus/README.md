# 채점표 코퍼스 (ADR-0002 원칙 ④)

`cargo test`(tests/corpus.rs)가 이 디렉터리를 릴리스 게이트로 돌린다.

## attacks/ — 미탐 채점표

실제 공급망 공격(주로 TeamPCP 캠페인)을 재현한 픽스처. **하나라도 놓치면 릴리스 불가.**

각 픽스처 = 저장소 모양의 디렉터리:

- `.github/workflows/*.yml` — 공격 시나리오 워크플로. **어떤 실제 사건을 재현했는지 주석 필수.**
- `expected.txt` — 반드시 탐지되어야 하는 규칙 ID (한 줄에 하나)
- `facts.txt` (선택) — 온라인 조회의 가짜 응답. 형식:
  - `resolve <repo@ref> <sha>` — 태그 해석 결과 (LOCK 대조용)
  - `reachable <sha> <true|false>` — 커밋 도달 가능성 (R5용)
  - `tags <repo> <개수>` — 버전 태그 수 (R2 교차 검증용)
  - `time <repo@ref> <unix초>` — 발행 시각 (R10용)
- `shield.lock` (선택) — 박제본이 필요한 시나리오(태그 이동)용

## benign/ — 오탐 채점표

실제 프로젝트의 워크플로를 본뜬 양성 픽스처. **🔴(High) 오탐이 하나라도 나오면 릴리스 불가.**
(🟡/🔵는 허용 — 오탐 0 보증은 빌드를 깨뜨리는 등급에만 적용된다, ADR-0002.)

각 워크플로에 무엇을 본떴는지 출처 주석을 남긴다.

## 추가 절차

1. 새 규칙을 만들면 그 규칙이 잡아야 할 공격 픽스처를 attacks/에 **반드시 함께** 추가한다.
2. 오탐 신고가 들어오면, 수정 전에 그 사례를 benign/에 먼저 추가한다 (회귀 방지).
3. 픽스처 주석에 재현한 사건/본뜬 출처를 남긴다 — 6개월 뒤의 우리가 "이게 왜 있지?"를 묻지 않도록.
