# just-shield

CI/CD 파이프라인이 소비하는 서드파티 도구·패키지의 공급망 오염을 방어하는 프로젝트. **북극성은 CI 자격증명(credential) 탈취 방지다.** 실존 위협 그룹 TeamPCP의 캠페인을 대표 검증 시나리오로 사용하되, 특정 그룹 전용이 아닌 일반 공급망 방어를 다룬다.

## Language

**TeamPCP**:
신뢰받는 오픈소스 도구의 배포 채널을 오염시켜 CI/CD 자격증명을 탈취하는 실존 위협 그룹(Google 추적명 UNC6780). 이 프로젝트에서는 방어 대상의 전부가 아니라 **대표 위협 시나리오**의 출처.
_Avoid_: "Team PCP를 막는 도구"라는 표현 — IOC 기반 전용 방어가 아님.

**공급망 공격 (Supply Chain Attack)**:
표적이 직접 공격받는 것이 아니라, 표적이 신뢰하고 소비하는 서드파티 산출물(패키지, GitHub Action, 컨테이너 이미지 등)이 오염되어 피해가 전파되는 공격.

**섭취 전 검증 (Pre-ingestion Verification)**:
파이프라인이 서드파티 산출물을 받아 실행하기 전에 "신뢰했던 그 산출물이 맞는가"를 확인하는 행위. 도구의 코어 방어 지점.

**검증 대상 (Verified Artifact)**:
v1에서는 워크플로 파일(`.github/workflows`)이 참조하는 GitHub Action. 패키지 락파일(v2 후보)과 바이너리/컨테이너 이미지는 현재 범위 밖.
_Avoid_: "의존성 스캐너" — 패키지 의존성은 아직 검사하지 않음.

## Relationships

- **공급망 공격**은 ① 업스트림 오염 → ② 오염된 산출물 섭취 → ③ CI 자격증명 수집 → ④ 외부 유출의 체인으로 진행된다
- **섭취 전 검증**은 이 체인의 ②를 차단한다 — ②가 막히면 ③④는 발생하지 않는다
- 자격증명 보호는 4개 층의 합이다: ⓐ 못 들어오게(섭취 검증) → ⓑ 훔칠 게 없게(피해 반경) → ⓒ 못 나가게(유출 차단 — 네이티브 egress 방화벽 등 플랫폼의 몫, 비범위) → ⓓ 가져가도 잡히게(카나리아 토큰, 보류)

**퍼스트파티 (First-party)**:
로컬 액션(`./...`)과 같은 소유자의 저장소가 제공하는 액션. 자기 코드의 연장이므로 섭취 검증 대상이 아니다.

**서드파티 (Third-party)**:
퍼스트파티가 아닌 모든 액션. GitHub 공식(`actions/*`)도 서드파티이나 완화된 심각도로 보고하고, 그 외에는 평판과 무관하게 예외 없이 엄격 적용한다 — TeamPCP가 오염시킨 것이 바로 보안 벤더들이었다.
_Avoid_: "신뢰할 만한 벤더" — 평판은 신뢰 근거가 아님.

**태그 하이재킹 (Tag Hijacking)**:
공격자가 기존 버전 태그를 악성 커밋으로 강제 이동시키는 기법. 가변 참조(태그/브랜치)를 쓰는 소비자만 피해를 입는다.

**임포스터 커밋 (Impostor Commit)**:
참조된 저장소의 정식 히스토리에 존재하지 않는 커밋. SHA로 핀했더라도 이것이면 오염 신호다.

**섭취 검증 규칙 (Tier 1)**:
"받는 산출물이 진짜이고 검증된 것인가"를 판정하는 규칙군 (R1~R5, R9, R10).

**쿨다운 (Cooldown)**:
발행된 지 일정 기간(기본 7일)이 지나지 않은 버전의 섭취를 경고하는 규칙. 제로데이를 탐지하는 게 아니라 미검증 기간을 회피하는 전략.
_Avoid_: "제로데이 탐지" — 탐지가 아니라 회피.

**피해 반경 규칙 (Tier 2)**:
오염을 못 막았을 때 "얼마나 털리는가"를 줄이는 규칙군 (R6~R8).
_Avoid_: 일반 워크플로 린트 규칙(인젝션 등) — 의도적 비범위, ADR-0001 참조.

**shield.lock (액션 락파일)**:
신뢰를 결정한 시점의 "태그 → SHA" 대응을 저장소 안에 박제한 파일. 이후 스캔은 현재 상태를 이 박제본과 대조해 태그 이동(하이재킹 신호)을 잡는다.
_Avoid_: "캐시" — 성능 장치가 아니라 신뢰의 기준점.

**심각도 등급 (Severity)**:
🔴 높음(사실 규칙 위반, 빌드 실패) / 🟡 중간(피해 확대 요인, 경고 — `--strict` 시 실패) / 🔵 안내(항상 경고만, 휴리스틱 단독 판정의 상한). ADR-0002의 사실 기반 원칙이 등급 배정을 지배한다.

**탈출구 (Escape Hatch)**:
경고를 의도적으로 수용하는 공식 통로 — 이유 필수 무시 주석, 신뢰 org 설정, `fix` 자동 SHA 교체. 탈출구 없는 경고는 도구를 꺼버리게 만든다.
_Avoid_: "예외 허용"을 비공식 우회(검사 자체를 끄기)로 해결하는 것.

**검사 엔진 (Scan Engine)**:
워크플로를 읽어 검증 판정을 내리는 코어 로직. CLI는 엔진의 1차 포장이며, GitHub Action·편집기 확장·에이전트 도구는 엔진을 호출하는 래퍼다.
_Avoid_: "액션을 만든다", "확장을 만든다" — 만드는 것은 엔진이고 그것들은 배포 형태.

## Flagged ambiguities

- "Team PCP 방어 프로그램"이 (A) 해당 그룹 전용 IOC 탐지인지 (B) 일반 공급망 방어인지 모호했음 — **해결: (B)**. TeamPCP 캠페인(Trivy/KICS/LiteLLM 오염, CI 자격증명 탈취, 2차 패키지 감염)은 검증 시나리오로 사용한다.
